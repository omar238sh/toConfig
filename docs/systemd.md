# Systemd Unit Files

The `toconfig::systemd` module generates systemd unit files (`.service`, `.socket`, `.timer`, `.mount`, `.automount`, `.path`, `.swap`) and drop-in overrides from typed Rust structs. All section structs implement `SystemdConfig` and are collected in a `SystemdUnit` (or a `DropIn` for overrides).

## SystemdUnit

`SystemdUnit` is the root container. It holds an ordered list of typed section structs and renders the complete unit file.

```rust
use toconfig::systemd::{SystemdConfig, SystemdUnit};

let unit = SystemdUnit::new()
    .with_comment("my-daemon.service — generated by toconfig")
    .push(/* any SystemdConfig section */)
    .push(/* ... */);

println!("{}", unit.generate());
```

## UnitSection

```rust
use toconfig::systemd::unit_section::UnitSection;

UnitSection::new()
    .description("My background daemon")
    .documentation("https://example.com/docs")
    .after("network-online.target")
    .requires("network-online.target")
    .wants("network.target")
    .before("multi-user.target")
    .conflicts("other.service")
    .condition_path_exists("/etc/my-daemon.conf")
```

## ServiceSection

```rust
use toconfig::systemd::service::{ServiceSection, ServiceType, Restart};

ServiceSection::new()
    .service_type(ServiceType::Simple)   // Simple | Forking | OneShot | Notify | Dbus | Idle
    .exec_start("/usr/bin/my-daemon --config /etc/my-daemon.conf")
    .exec_start_pre("/usr/bin/setup-script")
    .exec_stop("/usr/bin/my-daemon --stop")
    .exec_reload("/bin/kill -HUP $MAINPID")
    .restart(Restart::OnFailure)         // No | OnSuccess | OnFailure | OnAbnormal | Always
    .restart_sec("5s")
    .timeout_start_sec("30s")
    .timeout_stop_sec("30s")
    .environment("KEY=value")
    .environment_file("/etc/my-daemon.env")
    .working_directory("/var/lib/my-daemon")
    .user("my-daemon")
    .group("my-daemon")
    .runtime_directory("my-daemon")
    .state_directory("my-daemon")
    .private_tmp(true)
    .no_new_privileges(true)
    .protect_system("strict")
    .protect_home(true)
    .capabilities_bounding_set("")       // drop all capabilities
```

## TimerSection

```rust
use toconfig::systemd::timer::TimerSection;

TimerSection::new()
    .on_calendar("daily")              // calendar expression, e.g. "*-*-* 03:00:00"
    .on_boot_sec("5min")               // relative to boot
    .on_active_sec("1h")               // relative to last activation
    .on_unit_active_sec("1h")          // relative to last time the unit was active
    .persistent(true)                  // run missed triggers after system wakeup
    .randomized_delay_sec("10min")
    .unit("my-job.service")            // defaults to <timer-name>.service
    .accuracy_sec("1s")
```

## DropIn overrides

`DropIn` generates a drop-in override file (`/etc/systemd/system/foo.service.d/override.conf`). It holds the same section structs as `SystemdUnit`.

```rust
use toconfig::systemd::drop_in::DropIn;
use toconfig::systemd::service::{ServiceSection, Restart};

let drop_in = DropIn::new()
    .with_comment("override.conf — generated by toconfig")
    .push(
        ServiceSection::new()
            .restart(Restart::Always)
            .restart_sec("10s"),
    );

println!("{}", drop_in.generate());
```

## Generic sections and raw escape hatch

For any key not covered by the typed API, use `SystemdSection` / `SectionEntry`:

```rust
use toconfig::systemd::section::{SystemdSection, SectionEntry, RawSystemd};

let sec = SystemdSection::new("Service")
    .entry(SectionEntry::new("AmbientCapabilities", "CAP_NET_BIND_SERVICE"))
    .entry(SectionEntry::new("LockPersonality", "true"));

// Raw escape hatch
let raw = RawSystemd::new("[Service]\nOOMScoreAdjust=-500");
```

## End-to-end examples

### Simple service

```rust
use toconfig::systemd::{SystemdConfig, SystemdUnit};
use toconfig::systemd::unit_section::UnitSection;
use toconfig::systemd::service::{ServiceSection, ServiceType, Restart};
use toconfig::systemd::install::InstallSection;
use toconfig::output::ConfigOutput;

fn main() -> std::io::Result<()> {
    let unit = SystemdUnit::new()
        .with_comment("my-daemon.service — generated by toconfig")
        .push(
            UnitSection::new()
                .description("My background daemon")
                .after("network-online.target"),
        )
        .push(
            ServiceSection::new()
                .service_type(ServiceType::Simple)
                .exec_start("/usr/bin/my-daemon --config /etc/my-daemon.conf")
                .restart(Restart::OnFailure)
                .restart_sec("5s")
                .user("my-daemon")
                .private_tmp(true),
        )
        .push(InstallSection::new().wanted_by("multi-user.target"));

    ConfigOutput::at_path("/etc/systemd/system/my-daemon.service")
        .write(&unit)?;
    Ok(())
}
```

### Daily timer

```rust
use toconfig::systemd::{SystemdConfig, SystemdUnit};
use toconfig::systemd::unit_section::UnitSection;
use toconfig::systemd::timer::TimerSection;
use toconfig::systemd::install::InstallSection;
use toconfig::output::ConfigOutput;

fn main() -> std::io::Result<()> {
    let unit = SystemdUnit::new()
        .push(UnitSection::new().description("Daily backup timer"))
        .push(
            TimerSection::new()
                .on_calendar("*-*-* 03:00:00")
                .persistent(true)
                .randomized_delay_sec("15min"),
        )
        .push(InstallSection::new().wanted_by("timers.target"));

    ConfigOutput::at_path("/etc/systemd/system/backup.timer")
        .write(&unit)?;
    Ok(())
}
```
